# Reflection - Mission Tata Day 1

## What I Learned

This activity helped me understand the practical workflow followed by developers while working on collaborative projects. Previously, I was familiar with basic Git commands, but this task gave me hands-on experience with the complete GitHub contribution process.

## Key Takeaways

* Learned how to fork a repository and create my own copy.
* Understood the difference between local and remote repositories.
* Practiced cloning repositories and managing changes locally.
* Learned the importance of meaningful commit messages.
* Understood how Pull Requests are used for code review and collaboration.

## Challenges Faced

Initially, understanding the relationship between the original repository, forked repository, and local repository was slightly confusing. After practicing the workflow, I understood how changes move from a developer's machine to the main project repository.

## Conclusion

This task improved my confidence in using Git and GitHub. The workflow learned today will be useful for future projects, internships, and professional software development environments.
